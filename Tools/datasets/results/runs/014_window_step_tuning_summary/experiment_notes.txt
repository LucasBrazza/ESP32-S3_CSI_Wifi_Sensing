Experiment 014 - Window and step tuning

Description:
This experiment evaluates different sliding-window sizes and step sizes using the current temporal CSI feature set.

Method:
For each window/step pair, the feature dataset is regenerated and Top-K tuning is executed using file-based holdout validation. Fisher Score ranking is computed only on the training data.

Classifier:
- Model: decision_tree
- Max tree depth: 6
- Min samples split: 5

Top-K values tested for each window/step pair:
[30, 50, 70, 100, 126, 150, 180, 200, 231]

Window/step configurations tested:
- window_size=5, step_size=1
- window_size=5, step_size=2
- window_size=6, step_size=2
- window_size=8, step_size=2
- window_size=8, step_size=4
- window_size=10, step_size=2
- window_size=10, step_size=5
- window_size=12, step_size=3
- window_size=15, step_size=5

Best result:
- experiment_id: 014_w8_s2
- folder: 014_window8_step2_temporal_features
- window_size: 8
- step_size: 2
- samples: 417
- features per sample: 231
- top_k: 126
- accuracy: 0.657534
- macro F1-score: 0.667597
- weighted F1-score: 0.658344
- empty F1-score: 0.714286
- static_presence F1-score: 0.655172
- movement F1-score: 0.633333
- tree nodes: 47

Interpretation:
This experiment should be used to decide whether the current window configuration captures enough temporal information for movement recognition. If larger windows improve movement F1-score, the previous window was likely too short. If performance does not improve, the next step should be adding more motion-specific features or revising preprocessing.
