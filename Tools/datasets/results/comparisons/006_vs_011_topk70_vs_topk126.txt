Comparison - Experiment 006 vs Experiment 011

Objective:
Compare file-based holdout validation using top_k=70 and top_k=126.

Experiment 006:
- Name: 006_file_holdout_topk70_depth6_min5
- Validation: file-based stratified holdout 80/20
- Top-K features: 70
- Max tree depth: 6
- Min samples split: 5
- Accuracy: 0.551724
- Macro F1-score: 0.584280
- Weighted F1-score: 0.551952
- Tree nodes: 67

Experiment 011:
- Name: 011_file_holdout_topk126_depth6_min5
- Validation: file-based stratified holdout 80/20
- Top-K features: 126
- Max tree depth: 6
- Min samples split: 5
- Accuracy: 0.579310
- Macro F1-score: 0.585651
- Weighted F1-score: 0.572098
- Tree nodes: 59

Observed improvement:
- Accuracy increased by 0.027586
- Macro F1-score increased by 0.001371
- Weighted F1-score increased by 0.020146
- Tree size decreased from 67 nodes to 59 nodes

Per-class interpretation:
Using top_k=126 reduced the empty F1-score compared with top_k=70, but improved the balance of the presence-related classes. In experiment 011, static_presence reached F1-score 0.595041 and movement reached F1-score 0.514851, both better than in experiment 006.

Main limitation:
The movement class remains the main bottleneck. In experiment 011, 23 movement samples were classified as static_presence and 10 were classified as empty, while only 26 were correctly classified as movement.

Conclusion:
Experiment 011 should be considered the current best file-based decision tree configuration. The model improved compared with top_k=70, but the remaining limitation suggests that future improvements should focus on temporal features or window configuration to better distinguish static_presence from movement.
