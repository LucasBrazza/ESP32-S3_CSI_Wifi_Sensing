Comparison - Experiment 001 vs Experiment 003

Objective:
Compare the initial decision tree baseline with the tuned decision tree configuration.

Experiment 001:
- Name: 001_holdout_baseline_depth4_min2_top30
- Validation: stratified holdout 80/20
- Max tree depth: 4
- Min samples split: 2
- Accuracy: 0.582278
- Macro F1-score: 0.534149
- Weighted F1-score: 0.556645
- Empty recall: 0.230769
- Empty F1-score: 0.360000

Experiment 003:
- Name: 003_holdout_tuned_depth6_min5_top30
- Validation: stratified holdout 80/20
- Max tree depth: 6
- Min samples split: 5
- Accuracy: 0.620253
- Macro F1-score: 0.608970
- Weighted F1-score: 0.622193
- Empty recall: 0.564103
- Empty F1-score: 0.505747

Observed improvement:
- Accuracy increased by 0.037975
- Macro F1-score increased by 0.074821
- Weighted F1-score increased by 0.065548
- Empty recall increased by 0.333334
- Empty F1-score increased by 0.145747

Interpretation:
The tuned decision tree improved the overall classification performance and, more importantly, reduced the main limitation observed in the baseline model. In experiment 001, the classifier had difficulty identifying the empty class, frequently confusing it with static_presence. After tuning max_depth and min_samples_split, experiment 003 showed a considerable increase in empty recall, indicating that the classifier became better at recognizing windows without presence.

Embedded deployment consideration:
The tuned model is more complex than the baseline because it uses max_depth = 6 instead of max_depth = 4. However, this depth is still compatible with future embedded inference, since each prediction requires only a limited number of threshold comparisons. Therefore, the improvement in classification balance justifies the moderate increase in tree complexity.

Conclusion:
Experiment 003 should be considered the current best decision tree configuration for the next validation step.
